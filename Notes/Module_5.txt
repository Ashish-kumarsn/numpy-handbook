# Broadcasting


# Definition:
# NumPy mechanism that allows element-wise operations
# on arrays of different shapes.

# Features:
# 1. Automatic shape compatibility
# 2. No Python loops
# 3. Memory efficient
# 4. Virtual expansion (no actual copying)
# 5. Faster execution

# Common Uses:
# image + 20
# data / 255
# salary * 1.1
# X + bias

# Note:
# Broadcasting does NOT work for every shape.
# Arrays must satisfy broadcasting rules.

## What are these Rules 

# Broadcasting Rules

# Rule 1:
# Compare shapes from right to left.

# Rule 2:
# Dimensions are compatible if they are equal.

# Rule 3:
# Dimensions are also compatible if one of them is 1.

# Rule 4:
# Missing dimensions are treated as 1. 

# If arrays have different numbers of dimensions,
# NumPy pads the smaller shape with 1s from the LEFT
# before applying the broadcasting rules.

# Broadcasting fails when:
# Dimensions are different AND neither dimension is 1.

# Example:
# (2,3) + (3,)  -> Works because 
# (3,) becomes (1,3)

# (2,3) + (2,)  -> Error
# (2,) becomes (1,2)
# 3 vs 2 -> Not compatible

So whenever you apply any element wise Operations , the numpy first compare the shape 
By the rules , if it is passed then it will operate the operations element wise

Q1. What is virtual expansion?

Virtual expansion means NumPy behaves as if the smaller array were expanded, 
but it does not actually allocate memory for the expanded copy.

Q2. Why is broadcasting memory efficient?

Because NumPy does not duplicate data. It reuses the original values during computation.

Q3. Does broadcasting create the expanded array?

No. The expansion is conceptual only.
